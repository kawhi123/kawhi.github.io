滴答~滴

  1. 题目
    -… -.- -.-. - ..-. – .. … -.-.

  2. 解析
    知识点是摩斯密码(Morse Code)，它由两种基本信号和不同的间隔时间组成：短促的点信号“·”，读“滴”（Di）；保持一定时间的长信号“-”，读“嗒”（Da）。间隔时间：滴，1t；嗒，3t；滴嗒间，1t；字符间，3t；字间，7t。


    根据时间间隔将该电码分成-...，-.-，-.-.，-，..-.，--，..，...，-.-.九部分，分别是九个字母，对照摩斯密码表可以翻译得 BKCTFMISC ，所以该题答案为KEY{BKCTFMISC} 。也可以根据在线的摩斯密码翻译器进行翻译：https://www.mathsking.net/morse.htm

聪明的小羊

  1. 题目
    一只小羊翻过了2个栅栏
    KYsd3js2E{a2jda}

  2. 解析
    知识点是栅栏密码(Rail-fence Cipher)，就是把要加密的明文分成N个一组，然后把每组的第1个字连起来，形成一段无规律的话。 不过栅栏密码本身有一个潜规则，就是组成栅栏的字母一般不会太多。（一般不超过30个，也就是一、两句话）

根据题目信息该题是一个2栏栅栏密码，用上例方法得到KEY{sad23jjdsa2} 。也可以用在线的栅栏密码加解密工具：https://www.qqxiuzi.cn/bianma/zhalanmima.php

简单加密

  1. 题目
    e6Z9i~]8R~U~QHE{RnY{QXg~QnQ{^XVlRXlp^XI5Q6Q6SKY8jUAA

  2. 解析
    这题涉及到移位加密算法和base64加密。
    因为密文结尾是AA，可以联想到base64加密的==，关于base64加密的介绍笔者推荐这篇一篇博客：
    https://blog.csdn.net/qq_20545367/article/details/79538530


    此密文是通过移位加密后再base64加密，因此先要移位解密。根据ASCII码对照表可知，A和=对应的十进制数分别是65和61，因此将密文前移4位得到base64密文
    a2V5ezY4NzQzMDAwNjUwMTczMjMwZTRhNThlZTE1M2M2OGU4fQ==


    再通过base64在线工具解密得到
    key{68743000650173230e4a58ee153c68e8}


    另外也可以通过自己写一些脚本进行解密，比如python脚本，网上的教程也很多。

密码学

密码学概论

  • 待加密的消息称为明文（plaintext），它经过一个以密钥（key）为参数的函数变换，这个过程的结果就是所谓的密文（ciphertext）。
  • 破解密码的技术就是所谓的密码分析学（cryptanalysis），而他和设计密码的密码编码学（cryptography）合起来统称为密码学（cryptology）。
  • 用密钥K加密明文P得到密文C：
    解密密文C得到明文P：
    由此可以得到：
  • “让密码分析者知道加解密算法，并且把所有的秘密信息全部放在密钥中”这种思想称为Kerckhoff原理。

置换密码

  • 凯撒密码：明文按照字母顺序移动k个字母。
  • 采用“符号对符号”置换策略的通用系统称为单字母置换密码。

替代密码

  • 替代密码重新对字母表排序，但是并不伪装明文。

一次性交易

  • 一次性密钥：首先选择一个随机位串作为密钥，然后将明文转变成一个位串。最后，逐位计算这两个串的异或值，这样得到的密文是不可能被破解的。
    缺点：一次性密钥无法记忆，所以发送方和接收方必须随时携带书面的密钥副本。
  • 量子密码学

两个基本的密码学原则

  • 冗余度
    所有被加密的消息必须包含一定的冗余信息，也就是说，这些信息对于理解这些信息来说是不必要的。
  • 新鲜度
    必须采取某些措施确保每条接受到的消息可被验证是新鲜的，也就是说，它是最近发送出来的。
  • 密码学原则1：消息必须包含一定的冗余度。
    措施：CRC多项式编码
  • 密码学原则2：需要采用某种方法来对抗重放攻击。
    措施：在每条消息中包含一个仅在一段时间内有效的时间戳。

链接

  1. 行内式
    语法：[链接文字](链接地址 “title”)
    说明：title属性的效果是鼠标悬停在链接上时会指定出现的title文字，链接地址与title属性之间必须有一个空格！(title属性可加可不加)
    例如：
    欢迎来到[我的博客](https://kawhi123.github.io)
    欢迎来到[我的博客](https://kawhi123.github.io “kawhi”)
    显示效果：
    欢迎来到我的博客
    欢迎来到我的博客

  2. 参考式
    语法：[链接文字][链接标记]
    [链接标记]:链接地址 “title” 
    说明：链接标记需要占用一个独立的文本行！且不同链接的链接标记不能相同！
    例如：
    我觉得[谷歌][1]是个不错的搜索引擎。
    [1]:https://google.com
    显示效果：
    我觉得谷歌是个不错的搜索引擎。

  1. 自动链接
    语法：将网址的链接地址用<>包起来，Markdown就会自动把它转成链接。
    例如：<https://kawhi123.github.io>
    效果是 https://kawhi123.github.io

补充：Markdown能够自动识别网址链接，也就是说直接输入链接地址就可以跳转访问。所以写这么多好像并没有什么卵用，比如直接输入 https://kawhi123.github.io 。

Markdown基础语法

发表于 2018-11-23 | 更新于 2018-11-29 | 分类于 Markdown语法

标题

Markdown支持两种标题的语法，类Setext和类Atx形式。

  1. 类Setext形式是用=等号和-减号来表示(一个及以上），例如：
    这是一级标题
    =========
    这是二级标题
    ——————

  2. 类Atx形式是在行首插入1-6个#，对应一到六级标题，例如：
    # 这是一级标题
    ## 这是二级标题
    ###### 这是六级标题

    注意：#后有一个空格！！！

段落和换行

  1. 一个Markdown段落由一到多个连续的文本行组成，段落前后要有一个以上的空行（若某一行只包含空格和制表符也被视为空行）。

  2. 换行：在一个文本行末键入两个及以上的空格或制表符（Tab键）后回车（Enter键）即可换行。另外，由于Markdown兼容部分HTML语法，也可以用<br/>来换行。

文本样式

  1. 加粗：在需要加粗的文字前后各加两个*星号或者_底线符号，前后符号要匹配且符号和字之间不能有空格。例如：
    **hello world** 效果是 hello world

  2. 斜体：在需要倾斜的文字前后各加一个*星号或者_底线符号。例如：
    *Markdown* 效果是 Markdown

  3. 倾斜加粗：前后各加三个*星号或者_底线符号。例如：
    ***kawhi*** 效果是 kawhi

  4. 高亮：前后各加一个及以上的“`”反引号（Esc键下方）。例如：
    `#include<stdio.h>` 效果是 #include<stdio.h>

  5. 删除线：前后各加两个~波浪号（Esc键下方）。例如：
    ~~github~~ 效果是 github

  6. 分割线：在单独的一个文本行内用三个及以上的*星号、_底线符号来建立分割线（符号间可以有空格）。例如：
    * * *
    _ _ _

    效果都是

注意：一行内不得出现其他文字和符号!!!

列表

  1. 无序列表：每个文本行前使用*星号、+加号、-减号来标记。例如：
    + red
    + blue
    + yellow
    效果是

    • red
    • blue
    • yellow

    注意：“+”加号后有一个空格！！！

  2. 有序列表：使用阿拉伯数字接着一个英文句点。例如
    1. red
    2. blue
    3. yellow
    效果是

    1. red
    2. blue
    3. yellow

    注意：英文句点后有一个空格！！！

表格

  1. 每个表头或表格单元之间用“|”来隔开。表头下要有-|-|-，有几个表头就有几个“-”减号。

  2. 左对齐：默认左对齐，或在“-”左侧加“:”表示左对齐。例如：
    -|-|-

  3. 右对齐：在“-”右侧加“:”表示右对齐。例如：
    -:|-:|-:

  4. 居中对齐：在“-”两侧都加“:”表示居中对齐。例如：
    :-:|:-:|:-:

姓名|班级|学号
-|-|-
张三|1|10001
李四|2|10002
王五|3|10003

效果是

姓名 班级 学号
张三 1 10001
李四 2 10002
王五 3 10003

注意：表格前一定要有空行！！！

区块引用

  1. 基本使用：在需要引用的文本前加上>大于号，若要跳出引用空行即可。例如：
    >Markdown是一种可以使用普通文本编辑器编写的标记语言。
    效果是：

    Markdown是一种可以使用普通文本编辑器编写的标记语言。

  2. 嵌套使用：根据引用文本的层次加上不同数量的>。例如：
    >引用一
    >>引用二
    >>>引用三

    效果是：

    引用一

    引用二

    引用三

    补充：引用的区块内也可以使用其他的 Markdown 语法，包括标题、列表、代码区块等。

代码区块

  1. 每行代码前缩进四个空格或一个制表符即可建立代码区块。

  2. 在需要插入的代码区块的前一行及后一行各使用三个“`”反引号。例如：
    ```
    #include<stdio.h>
    void main()
    {
    printf(“hello world”);
    }
    ```
