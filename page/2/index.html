strcmp比较字符串

发表于 2019-02-22 | 更新于 2019-03-01 | 分类于 CTF ， BugkuCTF

ctf

thinking

die()函数：输出消息并终止当前脚本，相当于exit()

这题的关键是理解strcmp()字符串比较函数，和C语言中类似。strcmp()函数在进行字符串比较时，若str1和str2类型不同时会进行隐式转换，比如’123’ == 123，’1’ != 0。但这里有一个php版本漏洞，当待比较的数据是数组或对象时会出现错误。

官方文档说明

Note a difference between 5.2 and 5.3 versions
echo (int)strcmp(‘pending’,array());
will output -1 in PHP 5.2.16 (probably in all versions prior 5.3)
but will output 0 in PHP 5.3.3
Of course, you never need to use array as a parameter in string comparisions.

也就是说，在5.3之后的版本里，当我们传入的待比较的数据是数组或对象时，这个函数将发生错误，显示报错的警告信息后将return 0！！！

extract变量覆盖

发表于 2019-02-20 | 更新于 2019-03-01 | 分类于 CTF ， BugkuCTF

写在前面

代码审计的题都是php代码，每道题都涉及到一些函数，大多是函数的危险性引起的漏洞。这里有一个php官方文档，可以查阅很多函数的语法和功能。
http://php.net/manual/en/function.file-get-contents.php

ctf

flag{bugku-dmsj-p2sm3N}

thinking

  1. extract()函数从数组中将变量导入当前的符号表。该函数使用数组键名作为变量名，使用数组键值作为变量值。针对数组中的每个元素，将在当前符号表中创建对应的一个变量。该函数返回成功设置的变量数目。

  2. trim()函数移除字符串两侧的空白字符或其他预定义字符。

  3. file_get_contents()函数把整个文件读入一个字符串中。

tip

file_get_contents()函数作用的对象是一个文件，成功时返回文件中的字符串，失败则返回false。而该题作用的对象是一个变量$flag，因而返回bool型false。因此$content == false。因此只要构造$shiyan == $content就可以得到flag。

get

  1. extract()函数会覆盖已有的同名变量，具有危险性，慎用！

  2. $_GET变量传递的参数值都被封装到字符串中。还可以传递数组，具体见下一篇文章。

  3. ==弱类型比较时会发生隐式转换，若比较的值中有bool值时，另一个值就会转换为bool值。

  4. 在做代码审计题时，经常需要自己写php代码去理解和验证某个函数的功能，这是非常必要的。

get

X-Forwarded-For（XFF）是用来识别通过HTTP代理或负载均衡方式连接到Web服务器的客户端最原始的IP地址的HTTP请求头字段。如果没有XFF或者另外一种相似的技术，所有通过代理服务器的连接只会显示代理服务器的IP地址，而非连接发起的原始IP地址，这样的代理服务器实际上充当了匿名服务提供者的角色，如果连接的原始IP地址不可得，恶意访问的检测与预防的难度将大大增加。XFF的有效性依赖于代理服务器提供的连接原始IP地址的真实性，因此，XFF的有效使用应该保证代理服务器是可信的，比如可以通过创建可信服务器白名单的方式。

滴答~滴

  1. 题目
    -… -.- -.-. - ..-. – .. … -.-.

  2. 解析
    知识点是摩斯密码(Morse Code)，它由两种基本信号和不同的间隔时间组成：短促的点信号“·”，读“滴”（Di）；保持一定时间的长信号“-”，读“嗒”（Da）。间隔时间：滴，1t；嗒，3t；滴嗒间，1t；字符间，3t；字间，7t。


    根据时间间隔将该电码分成-...，-.-，-.-.，-，..-.，--，..，...，-.-.九部分，分别是九个字母，对照摩斯密码表可以翻译得 BKCTFMISC ，所以该题答案为KEY{BKCTFMISC} 。也可以根据在线的摩斯密码翻译器进行翻译：https://www.mathsking.net/morse.htm

聪明的小羊

  1. 题目
    一只小羊翻过了2个栅栏
    KYsd3js2E{a2jda}

  2. 解析
    知识点是栅栏密码(Rail-fence Cipher)，就是把要加密的明文分成N个一组，然后把每组的第1个字连起来，形成一段无规律的话。 不过栅栏密码本身有一个潜规则，就是组成栅栏的字母一般不会太多。（一般不超过30个，也就是一、两句话）

根据题目信息该题是一个2栏栅栏密码，用上例方法得到KEY{sad23jjdsa2} 。也可以用在线的栅栏密码加解密工具：https://www.qqxiuzi.cn/bianma/zhalanmima.php

简单加密

  1. 题目
    e6Z9i~]8R~U~QHE{RnY{QXg~QnQ{^XVlRXlp^XI5Q6Q6SKY8jUAA

  2. 解析
    这题涉及到移位加密算法和base64加密。
    因为密文结尾是AA，可以联想到base64加密的==，关于base64加密的介绍笔者推荐这篇一篇博客：
    https://blog.csdn.net/qq_20545367/article/details/79538530


    此密文是通过移位加密后再base64加密，因此先要移位解密。根据ASCII码对照表可知，A和=对应的十进制数分别是65和61，因此将密文前移4位得到base64密文
    a2V5ezY4NzQzMDAwNjUwMTczMjMwZTRhNThlZTE1M2M2OGU4fQ==


    再通过base64在线工具解密得到
    key{68743000650173230e4a58ee153c68e8}

thinking

  1. error_reporting(0) : 关闭所有错误报告。

  2. include （或 require）语句会获取指定文件中存在的所有文本/代码/标记，并复制到使用 include 语句的文件中。

  3. isset — 检测变量是否已设置并且非 NULL。

  4. preg_match : 执行匹配正则表达式。
    /*****/斜杠内表示的是正则表达式，^表示以该字符串为开头，$表示以该字符串为结尾，\w一般都是指包含大小写字母数字和下划线 相当于([0-9a-zA-Z])，+表示该字符串出现一次或多次。

  5. var_dump() 函数显示关于一个或多个表达式的结构信息，包括表达式的类型与值。数组将递归展开值，通过缩进显示其结构。

  6. eval() 函数把字符串按照 PHP 代码来计算。该字符串必须是合法的 PHP 代码，且必须以分号结尾。如果没有在代码字符串中调用 return 语句，则返回 NULL。如果代码中存在解析错误，则 eval() 函数返回 false。

  7. $和$$
    $args(一个美元)是一个普通变量，名称为：args，存储任何值，如：string，integer，float等。
    $$var(两个美元)是一个可变变量，一个可变变量获取了一个普通变量的值作为这个可变变量的变量名。

  8. $GLOBALS — 引用全局作用域中可用的全部变量。
    $GLOBALS是一个超全局变量，包含了全部变量的全局组合数组。在$GLOBALS 数组中，每一个变量为一个元素，键名对应变量名，值对应变量的内容。
