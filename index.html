X-Forwarded-For（XFF）是用来识别通过HTTP代理或负载均衡方式连接到Web服务器的客户端最原始的IP地址的HTTP请求头字段。如果没有XFF或者另外一种相似的技术，所有通过代理服务器的连接只会显示代理服务器的IP地址，而非连接发起的原始IP地址，这样的代理服务器实际上充当了匿名服务提供者的角色，如果连接的原始IP地址不可得，恶意访问的检测与预防的难度将大大增加。XFF的有效性依赖于代理服务器提供的连接原始IP地址的真实性，因此，XFF的有效使用应该保证代理服务器是可信的，比如可以通过创建可信服务器白名单的方式。

strcmp比较字符串

ctf

thinking

die()函数：输出消息并终止当前脚本，相当于exit()

这题的关键是理解strcmp()字符串比较函数，和C语言中类似。strcmp()函数在进行字符串比较时，若str1和str2类型不同时会进行隐式转换，比如’123’ == 123，’1’ != 0。但这里有一个php版本漏洞，当待比较的数据是数组或对象时会出现错误。

官方文档说明

Note a difference between 5.2 and 5.3 versions
echo (int)strcmp(‘pending’,array());
will output -1 in PHP 5.2.16 (probably in all versions prior 5.3)
but will output 0 in PHP 5.3.3
Of course, you never need to use array as a parameter in string comparisions.

也就是说，在5.3之后的版本里，当我们传入的待比较的数据是数组或对象时，这个函数将发生错误，显示报错的警告信息后将return 0！！！

md5()函数

ctf

thinking

要解这道题必须要理解三点：

  • isset()函数：确定是否设置了变量（这个函数非常常见）。
  • md5()函数：计算字符串的md5哈希值。通俗地讲就是就是对字符串进行md5加密。
  • 强等于（===）和弱等于（==）：强等于要求值和类型都要相等且不会进行类型转换，弱等于只要求值相等，也就是说类型可以不同，但类型转换后的值要相等。

要得到flag必须满足三个条件：

  • 必须使用$_GET传递参数username和password
  • ‘username’ != ‘password’
  • md5(‘username’) === md5 (‘password’)

tip

在php中，md5()函数是不能处理数组的，也就是说md5(a[]) == NULL。而URL可以传递数组参数，于是这题就迎刃而解了。

如果对md5加密比较了解的话，可能会想到md5加密本身的缺陷，但是这题是强等于（===），所以不能用Hash比较缺陷解题。有关弱类型比较及Hash比较缺陷见“md5加密相等绕过”。

Linux简介

  • Linux是众多操作系统之一，操作系统（Operating System，简称OS）是软件的一部分，它是计算机硬件基础上的第一层软件，是硬件和其他软件沟通的桥梁（接口）。

  • Linux只是一个内核，内核指的是一个提供设备驱动、文件系统、进程管理、网络通信等功能的系统软件，内核并不是一套完整的操作系统，它只是操作系统的核心。一些组织或厂商将Linux内核与各种软件和文档包装起来，并提供系统安装界面和系统配置、设定与管理工具，就构成了Linux的发行版本。因此把Red Hat、Ubuntu、CentOS等直接说成Linux是不确切的，它们是Linux的发行版本，更确切地说是“以Linux为核心的操作系统软件包”。

Linux命令基本格式

  1. 命令提示符
    [kawhi@localhost ~]$

    • []：这是提示符的分隔符号，没有特殊含义。
    • kawhi/root：显示的是当前的登录用户。
    • @：分隔符号，没有特殊含义。
    • localhost：当前系统的简写主机名（完整主机名是 localhost.localdomain）。
    • ~：代表用户当前所在的目录，此例中用户当前所在的目录是家目录。
    • $：命令提示符，Linux 用这个符号标识登录的用户权限等级。如果是超级用户，提示符就是 #；如果是普通用户，提示符就是 $。

    如果切换用户所在目录，那么命令提示符中的会变成用户当前所在目录的最后一个目录（不显示完整的所在目录 /usr/ local/，只显示最后一个目录 local)。
    [root@localhost kawhi]# cd /usr/local
    [root@localhost local]#

  2. 命令基本格式
    命令[选项][参数]

    • 选项的作用是调整命令功能。如果没有选项，那么命令只能执行最基本的功能；而一旦有选项，则可以显示更加丰富的数据。
      Linux 的选项又分为短格式选项（-）和长格式选项（–）。短格式选项是英文的简写，用一个减号调用，而长格式选项是英文完整单词，一般用两个减号调用。

    • 参数是命令的操作对象，一般文件、目录、用户和进程等可以作为参数被命令操作。命令一般都需要加入参数，用于指定命令操作的对象是谁。如果可以省略参数，则一般都有默认参数。

Linux文件权限

  1. 文件的所有者、所属组和其他人

    • 用户组就是一组用户的集合
    • 文件的所有者就是这个文件的建立者，而系统中绝大多数系统文件都是由 root 建立的，所以大多数系统文件的所有者都是 root。

  2. Linux权限位
    drwxr-xr-x

    第1位代表文件类型，Linux 不像 Windows 使用扩展名表示文件类型，而是使用权限位的第 1 位表示文件类型。

    • “-“：普通文件。
    • “b”：块设备文件。这是一种特殊设备文件，存储设备都是这种文件，如分区文件 /dev/sda1 就是这种文件。
    • “c”：字符设备文件。这也是特殊设备文件，输入设备一般都是这种文件，如鼠标、键盘等。
    • “d”：目录文件。Linux 中一切皆文件，所以目录也是文件的一种。
    • “l”：软链接文件。
    • “p”：管道符文件。这是一种非常少见的特殊设备文件。
    • “s”：套接字文件。这也是一种特殊设备文件，一些服务支持 Socket 访问，就会产生这样的文件。

    2~4位代表文件所有者的权限。

    • r：代表 read，是读取权限。
    • w：代表 write，是写权限。
    • x：代表 execute，是执行权限。

    如果有字母，则代表拥有对应的权限；如果是”-“，则代表没有对应的权限。

    第 5~7 位代表文件所属组的权限，同样拥有”rwx”权限。

    第 8~10 位代表其他人的权限，同样拥有”rwx”权限。

绝对路径和相对路径

  1. 绝对路径：Linux 的最高目录是根目录。如果路径是从根目录开始，一级一级指定的，那使用的就是绝对路径。

  2. 所谓相对路径，就是只从当前所在目录开始，切换目录。

Linux中一切皆文件

“一切皆文件”，指的是，对所有文件（目录、字符设备、块设备、套接字、打印机等）操作，读写都可用fopen()/fclose()/fwrite()/fread()等函数进行处理。屏蔽了硬件的区别，所有设备都抽象成文件，提供统一的接口给用户。虽然类型各不相同，但是对其提供的却是同一套操作界面。更进一步，对文件的操作也可以跨文件系统执行。

序列化和反序列化

在学习PHP反序列化漏洞之前，我们必须明白什么是序列化和反序列化，这就要涉及到两个函数：

  1. serialize()：将一个对象转换成一个可传输的字符串。

  2. unserialize()：将序列化的字符串还原成相应的对象，反序列化是序列化的逆过程。

简单的理解：序列化就是使用serialize()将对象的用字符串的方式进行表示，反序列化是使用unserialize()将序列化的字符串，构造成相应的对象，反序列化是序列化的逆过程。序列化的对象可以是class也可以是Array,string等其他对象。

对象序列化和反序列化的功能

  1. 对象序列化的功能
    对象是在内存中存储的数据类型，寿命通常随着生成该对象的程序的终止而终止，但是有些情况下需要将对象的状态保存下来，然后在需要使用的时候将对象恢复，对象状态的保存操作就是对象序列化的过程。对象序列化可以将对象的状态通过数值和字符记录下来，以某种存储形式使自定义对象持久化，方便需要时候将对象进行恢复使用，用于对象的传递以及使程序代码更具维护性。

  2. 对象反序列化的功能
    把序列化后的字符串转化为对象，恢复原本对象后用于程序或代码的各种操作。
