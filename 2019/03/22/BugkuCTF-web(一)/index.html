域名解析

ctf

听说把 flag.baidu.com 解析到123.206.87.240 就能拿到flag

thinking

CTF题目可以说是非常关键了，开门见山告诉我们此题要将域名解析到指定IP，google一下（搜索引擎的使用很重要）。大浪淘沙找到了一篇有用的文章：https://www.cnblogs.com/cl-blogs/p/4160483.html （p.s.：博客的动态背景挺好看的）。提取有用信息：修改hosts文件!

tip

  1. 找到hosts文件的位置：C:\Windows\System32\drivers\etc\hosts

    • 用记事本打开添加单独一行（注意IP地址前不要有空格，IP地址和域名之间至少一个空格！）
      123.206.87.240 flag.baidu.com

    • 系统提示我没有管理员权限不能修改此文件，因此要获取管理员权限。网上说win+R唤出运行窗口，输入gpedit.msc打开了组策略编辑器，然后biabiabia一顿操作。（反正我没成功，所以懒得写）如果成功了就可以修改hosts文件。

  3. （2行不通的话）

    • 管理员身份运行cmd（我用的是替代工具cmder）：找到cmd.exe文件右键以管理员身份运行。（百度上方法挺多的，用的也比较多）

    • cd切换到hosts文件的路径，ls查看目录下文件，vi hosts进入编辑模式即可修改。（这里需要学习cmd的一些常用命令，vi/vim命令非常重要）

  4. 网页输入flag.baidu.com即可拿到flag。

    • 或者命令行输入curl flag.baidu.com直接查看网页内容。

    • cat hosts可以查看文件内容。

get

  1. hosts是一个没有扩展名的系统文件，可以用记事本等工具打开，其作用就是将一些常用的网址域名与其对应的IP地址建立一个关联“数据库”，当用户在浏览器中输入一个需要登录的网址时，系统会首先自动从Hosts文件中寻找对应的IP地址，一旦找到，系统会立即打开对应网页，如果没有找到，则系统会再将网址提交DNS域名解析服务器进行IP地址的解析。
    需要注意的是，Hosts文件配置的映射是静态的，如果网络上的计算机更改了请及时更新IP地址，否则将不能访问。

  2. curl命令详解：https://www.jianshu.com/p/07c4dddae43a

  3. cat命令：https://www.cnblogs.com/mengchunchen/p/8066306.html

thinking

打开链接发现页面不停地刷新，查看源代码可以发现内嵌了一个<script>标签。

setTimeout()用于设置一个定时器，reload()方法用于刷新当前页面。因此通过浏览器禁用JavaScript可以得到flag。

源代码最后有一句flag is here，通过抓包应该也能找到flag。

tip

方法一：禁用JavaScript

  1. 打开Google首页，点击右上角的自定义及控制Google Chrome按钮，进入设置>高级>内容设置>Javascript，将其屏蔽。

  2. 刷新题目的网页，当图片加载出来后查看网页源代码（Ctrl+U）,就可以拿到flag：flag{dummy_game_1s_s0_popular}。

方法二：Burp Suite抓包

  1. 安装JDK并配置java环境

    网上教程很多，但是大多需要手动配置环境变量，我找到了一个比较简单的方法可以跳过这一步骤。

    • https://www.java.com/en/download/win10.jsp
      进入链接点击Agree and Start Free Download下载应用程序JavaSetup8u201。

    • 点击该应用程序按照提示进行安装（可以更改安装路径），安装结束后就OK了，不需要再配置环境变量。

    • 进入cmd窗口输入指令java -version或者可验证是否安装成功。

  2. 下载Burp Suite 1.7.31破解版并安装

    官网有付费正版，土豪随意。下面是破解版的安装步骤。

    • https://www.52pojie.cn/thread-691448-1-1.html
      这个帖子简直太赞了，附有百度云盘下载链接和提取码，而且有多个版本的burp。

    • 安装有些复杂。双击burp-loader-keygen文件会弹出一个破解界面，将License Text改为www.52pojie.cn,复制下面的License。

    • 点击Run，这时会弹出一个 Enter license key 的安装界面，粘贴刚复制的License。

    • 点击Next>Manual activation>Copy request。粘贴到破解界面中的Activation Request，这时会得到Activation Response，将其粘贴到安装界面中的Paste response，最后点击Next就破解并安装完成了。

thinking

  1. error_reporting(0) : 关闭所有错误报告。

  2. include （或 require）语句会获取指定文件中存在的所有文本/代码/标记，并复制到使用 include 语句的文件中。

  3. isset — 检测变量是否已设置并且非 NULL。

  4. preg_match : 执行匹配正则表达式。
    /*****/斜杠内表示的是正则表达式，^表示以该字符串为开头，$表示以该字符串为结尾，\w一般都是指包含大小写字母数字和下划线 相当于([0-9a-zA-Z])，+表示该字符串出现一次或多次。

  5. var_dump() 函数显示关于一个或多个表达式的结构信息，包括表达式的类型与值。数组将递归展开值，通过缩进显示其结构。

  6. eval() 函数把字符串按照 PHP 代码来计算。该字符串必须是合法的 PHP 代码，且必须以分号结尾。如果没有在代码字符串中调用 return 语句，则返回 NULL。如果代码中存在解析错误，则 eval() 函数返回 false。

  7. $和$$
    $args(一个美元)是一个普通变量，名称为：args，存储任何值，如：string，integer，float等。
    $$var(两个美元)是一个可变变量，一个可变变量获取了一个普通变量的值作为这个可变变量的变量名。

  8. $GLOBALS — 引用全局作用域中可用的全部变量。
    $GLOBALS是一个超全局变量，包含了全部变量的全局组合数组。在$GLOBALS 数组中，每一个变量为一个元素，键名对应变量名，值对应变量的内容。

thinking

打开链接后提示网站存在漏洞。猜测网页后门可能存在漏洞。

tip

  1. 使用御剑扫一下网页后台，看能不能扫到webshell。
    这里推荐一个CTF工具库：https://www.ctftools.com/down/
    复制域名到御剑，点击开始扫描可以得到两个网页地址。

  2. 点击进入第二个网页（webshell），发现需要密码。

  3. 再使用burp暴力破解密码。

  4. 鼠标单击pass=4fahg所在的那一行，右键选择 Send to Itruder。
    然后依次点击 Intruder>Payloads，Payload type选择Simple list，Add from list选择Passwords。
    最后点击右上角Start attack，开始暴力破解密码。

  5. 点击Length按照长度排序，发现有一个不同的长度是1110，对应的Payload是 hack。

  6. 最后输入破解的密码 hack 可以得到 flag{hack_bug_ku035}。

打开链接看到一个管理员系统的提交界面，管理员的Username一般是admin。
查看源代码会发现有一段注释掉了的base64密文：dGVzdDEyMw==（隐藏很深，要一直往下翻），解密得到 text123，这应该就是Password。
提交发现：

最后一句话提示需要使用本地ip进行提交，因此想到了X-Forwarded-For（XFF）。

tip

打开burp suite进行抓包，在原始请求头里添加一行键值对：X-Forwarded-For: 127.0.0.1。（127.0.0.1是本地ip）

得到 flag{85ff2ee4171396724bae20c0bd851f6b}。

get

X-Forwarded-For（XFF）是用来识别通过HTTP代理或负载均衡方式连接到Web服务器的客户端最原始的IP地址的HTTP请求头字段。如果没有XFF或者另外一种相似的技术，所有通过代理服务器的连接只会显示代理服务器的IP地址，而非连接发起的原始IP地址，这样的代理服务器实际上充当了匿名服务提供者的角色，如果连接的原始IP地址不可得，恶意访问的检测与预防的难度将大大增加。XFF的有效性依赖于代理服务器提供的连接原始IP地址的真实性，因此，XFF的有效使用应该保证代理服务器是可信的，比如可以通过创建可信服务器白名单的方式。
