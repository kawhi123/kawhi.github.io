extract变量覆盖

ctf

flag{bugku-dmsj-p2sm3N}

thinking

  1. extract()函数从数组中将变量导入当前的符号表。该函数使用数组键名作为变量名，使用数组键值作为变量值。针对数组中的每个元素，将在当前符号表中创建对应的一个变量。该函数返回成功设置的变量数目。

  2. trim()函数移除字符串两侧的空白字符或其他预定义字符。

  3. file_get_contents()函数把整个文件读入一个字符串中。

tip

file_get_contents()函数作用的对象是一个文件，成功时返回文件中的字符串，失败则返回false。而该题作用的对象是一个变量$flag，因而返回bool型false。因此$content == false。因此只要构造$shiyan == $content就可以得到flag。

get

  1. extract()函数会覆盖已有的同名变量，具有危险性，慎用！

  2. $_GET变量传递的参数值都被封装到字符串中。还可以传递数组。具体见“strcmp比较字符串”。

  3. ==弱类型比较时会发生隐式转换，若比较的值中有bool值时，另一个值就会转换为bool值。

  4. 在做代码审计题时，经常需要自己写php代码去理解和验证某个函数的功能，这是非常必要的。

strcmp比较字符串

ctf

thinking

die()函数：输出消息并终止当前脚本，相当于exit()

这题的关键是理解strcmp()字符串比较函数，和C语言中类似。strcmp()函数在进行字符串比较时，若str1和str2类型不同时会进行隐式转换，比如’123’ == 123，’1’ != 0。但这里有一个php版本漏洞，当待比较的数据是数组或对象时会出现错误。

官方文档说明

Note a difference between 5.2 and 5.3 versions
echo (int)strcmp(‘pending’,array());
will output -1 in PHP 5.2.16 (probably in all versions prior 5.3)
but will output 0 in PHP 5.3.3
Of course, you never need to use array as a parameter in string comparisions.

也就是说，在5.3之后的版本里，当我们传入的待比较的数据是数组或对象时，这个函数将发生错误，显示报错的警告信息后将return 0！！！

tip

而$_GET只能传递字符串和数组类型的变量，因此构造一个数组类型的变量，利用这个漏洞就可以得到flag。

urldecode二次编码绕过

ctf

thinking

  1. eregi()函数：不区分大小写的正则表达式匹配。ereg()函数则是区分大小写的正则表达式匹配。

  2. urldecode()函数：解码URL编码的字符串。
    Warning：superglobals $_GET和$_REQUEST已经被解码。在$_GET和$_REQUEST的元素上使用urldecode()可能会产生危险性！

  3. 代码解读：通过$_GET传递的id不能有”hackerDJ”字段，而解码后的id又要 == “hackerDJ”。另外$_GET默认会对URL中的字符进行解码，但URL中的数字和字母默认是不解码的，但有其对应的URL编码。

tip

这题很容易想到对”hackerDJ”进行两次编码，然后通过$_GET传递给id，这样会默认进行一次解码，显然不会有”hackerDJ”字段。再经过urldecode()函数二次解码得到”hackerDJ”，从而得到flag。

get

  1. 了解URL编码和解码。
  2. urldecode()函数具有危险性。

md5()函数

ctf

thinking

要解这道题必须要理解三点：

  • isset()函数：确定是否设置了变量（这个函数非常常见）。
  • md5()函数：计算字符串的md5哈希值。通俗地讲就是就是对字符串进行md5加密。
  • 强等于（===）和弱等于（==）：强等于要求值和类型都要相等且不会进行类型转换，弱等于只要求值相等，也就是说类型可以不同，但类型转换后的值要相等。

要得到flag必须满足三个条件：

  • 必须使用$_GET传递参数username和password
  • ‘username’ != ‘password’
  • md5(‘username’) === md5 (‘password’)

tip

在php中，md5()函数是不能处理数组的，也就是说md5(a[]) == NULL。而URL可以传递数组参数，于是这题就迎刃而解了。

如果对md5加密比较了解的话，可能会想到md5加密本身的缺陷，但是这题是强等于（===），所以不能用Hash比较缺陷解题。有关弱类型比较及Hash比较缺陷见“md5加密相等绕过”。

md5加密相等绕过

ctf

thinking

  1. @ 是php中的一个错误运算控制符。当将其放置在一个php表达式之前，该表达式可能产生的任何错误信息都将被忽略掉。

  2. === 在进行比较的时候，会先判断两种字符串的类型是否相等，再比较
    == 在进行比较的时候，会先将字符串类型转化成相同，再比较
    如果比较一个数字和字符串或者比较涉及到数字内容的字符串，则字符串会被转换成数值并且比较按照数值来进行

  3. 代码解读
    首先要用$_GET传递参数a，a的值不等于’QNKCDZO’且md5加密后的’QNKCDZO’和a要弱相等。

tip

当一个字符串被当作一个数值来取值，其结果和类型如下：如果该字符串没有包含’.’,’e’,’E’并且其数值值在整形的范围之内，该字符串被当作int来取值，其他所有情况下都被作为float来取值，该字符串的开始部分决定了它的值，如果该字符串以合法的数值开始，则使用该数值，否则其值为0。

“0e1247363” == “0e2754217”相比较的时候，会将0e这类字符串识别为科学技术法的数字，0的无论多少次方都是零，所以相等。

这里介绍一批字符串经过md5加密后是以0e开头的，0e开头的字符串在比较的时候会将其视作为科学计数法，所以无论0e后面是什么，0的多少次方还是0。

所以这题URL传参?a=240610708就可以成功绕过得到flag{bugku-dmsj-am9ls}。这题利用的是Hash比较缺陷。

ref

php中md5函数漏洞：https://blog.csdn.net/vspiders/article/details/78218512
php弱类型比较：https://www.cnblogs.com/Mrsm1th/p/6745532.html

数组返回NULL绕过

ctf

thinking

  1. 代码解读
    $_GET传入的password的值经ereg()函数必须是字母或数字，但是经strpos()函数又必须包含--，看上去有些矛盾。

  2. ereg()函数：区分大小写的正则匹配，待匹配的数据类型必须是字符串，若是数组类型将会报错。

也就是说当传递的是数组时，ereg()函数返回值是NULL，在强比较（===）时不发生隐式转换，因此NULL !== false，也就是说该题传递数组时可以绕过第一个if判断。

  1. strpos()函数：查找字符串中第一次出现子字符串的位置（区分大小写）。找到则返回子字符串第一出现的位置（注意字符串从0开始），找不到则返回bool值false。

因此当传递数组时，strpos()函数返回值也是NULL。所以这题传递数组就可以拿到flag。

tip

方法一：传递数组返回NULL绕过
?password[]

方法二：%00截断
%00是URL编码，对应的十六进制ASCII码是0x00，也就是空字符（NULL）。有些函数在处理时会将其视为字符串的结束符，也就是会发生截断。
?password=123%00--
